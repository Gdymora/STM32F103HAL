 Регістри в блоці **RCC** йдуть один за одним з кроком у **4 байти** (оскільки кожен регістр 32-бітний).

Ось повна карта регістрів **RCC** для STM32F103 (основні, які використовуються найчастіше):

### Карта регістрів RCC (Base: 0x4002 1000)

| Зміщення (Offset) | Назва регістра | Функція (що він робить) |
| :--- | :--- | :--- |
| **0x00** | **RCC_CR** | **Clock Control.** Вмикає генератори (HSI, HSE, PLL). Тут ми чекаємо, поки кварц "заведеться". |
| **0x04** | **RCC_CFGR** | **Clock Configuration.** Вибирає джерело тактування (звідки беремо частоту) та дільники для шин. |
| **0x08** | **RCC_CIR** | **Clock Interrupt.** Керування перериваннями від системи тактування (наприклад, якщо кварц зламався). |
| **0x0C** | **RCC_APB2RSTR** | **APB2 Peripheral Reset.** Дозволяє примусово скинути (reset) модулі на шині APB2 (GPIO, ADC). |
| **0x10** | **RCC_APB1RSTR** | **APB1 Peripheral Reset.** Скидання модулів на повільній шині APB1 (Timers, UART2, CAN). |
| **0x14** | **RCC_AHBENR** | **AHB Enabler.** Вмикає тактування для DMA та Flash-пам'яті. |
| **0x18** | **RCC_APB2ENR** | **APB2 Enabler.** Твій улюблений регістр! Вмикає тактування GPIOA, B, C, D та ADC. |
| **0x1C** | **RCC_APB1ENR** | **APB1 Enabler.** Вмикає тактування для таймерів, UART, I2C, SPI2. |
| **0x20** | **RCC_BDCR** | **Backup Domain Control.** Керування годинником реального часу (RTC) та LSE. |
| **0x24** | **RCC_CSR** | **Control/Status.** Статус скидання (якщо чип перезавантажився від кнопки або помилки). |

---



### Як це "прочитати" просторово?

Уявіть, що процесор — це велике місто, а **RCC** — це **Головна Електростанція**.

1.  **RCC_CR (0x00)** — Це головні турбіни. Ви вирішуєте: працюємо від внутрішнього генератора чи від зовнішнього кварцу.
2.  **RCC_CFGR (0x04)** — Це коробка передач. Ви вирішуєте: чи буде процесор працювати на 8 МГц, чи розгонимо його до 72 МГц.
3.  **RCC_APB2ENR (0x18)** — Це щитова з рубильниками для окремих районів (модулів). Кожен біт — це окремий рубильник. 
    *   Біт 2 — рубильник для району "Порт А".
    *   Біт 4 — рубильник для району "Порт С".

### Чому в коді ми додаємо саме `0x18`?
Тому що ви хочете подати енергію на порти GPIO. А рубильники для GPIO знаходяться саме в 7-й за рахунком "кімнаті" (якщо рахувати від нуля по 4 байти: $0, 4, 8, 12, 16, 20, 24$... о стоп, $0x18$ — це і є 24 в десятковій системі).

**Порада для малювання:**
Намалюйте великий прямокутник **RCC_BASE (0x4002 1000)**. Всередині нього намалюйте "комірки" по 4 байти. Зафарбуйте комірку за адресою `+0x18` яскравим кольором і підпишіть: **"Тут вмикається життя портів"**.

Чи хочеш, щоб я розписав так само детально всі "кімнати" всередині самого порту **GPIOA** (від CRL до ODR)?
Звісно! У мікроконтролері STM32 величезна кількість блоків, і кожен має свою "адресну територію". Якщо **RCC** — це "електростанція", то інші блоки — це заводи, пошти та вокзали.

Для вашого навчання зараз найважливішими є блоки **GPIO** (General Purpose Input/Output). Ви вже почали з ними працювати, але давайте подивимося на їхню повну карту регістрів.

У кожного порту (GPIOA, GPIOB, GPIOC...) набір регістрів **однаковий**, змінюється лише базова адреса.

### Карта регістрів GPIO (на прикладі GPIOA: 0x4001 0800)

У кожному порту "кімнати" (регістри) йдуть з кроком 4 байти:

| Зміщення (Offset) | Назва регістра | Що він робить (просторово) |
| :--- | :--- | :--- |
| **0x00** | **GPIOx_CRL** | **Control Low.** Налаштування пінів **0–7**. (4 біти на пін). |
| **0x04** | **GPIOx_CRH** | **Control High.** Налаштування пінів **8–15**. (4 біти на пін). |
| **0x08** | **GPIOx_IDR** | **Input Data.** Регістр "читання". Тут ми дивимось, чи натиснута кнопка (0 чи 1). |
| **0x0C** | **GPIOx_ODR** | **Output Data.** Регістр "запису". Сюди ми пишемо 1, щоб засвітити LED. |
| **0x10** | **GPIOx_BSRR** | **Bit Set/Reset.** "Розумний" спосіб миттєво ввімкнути або вимкнути пін. |
| **0x14** | **GPIOx_BRR** | **Bit Reset.** Окремий регістр тільки для вимкнення пінів. |
| **0x18** | **GPIOx_LCKR** | **Configuration Lock.** Дозволяє "заморозити" налаштування, щоб код випадково їх не змінив. |

### Які ще блоки існують в STM32?

Окрім RCC та GPIO, у вашому "місті-мікроконтролері" є багато інших важливих департаментів. Ось основні групи, які ви будете вчити далі:

#### 1. Системні блоки (Керування мізками)
*   **NVIC (Nested Vectored Interrupt Controller):** Величезний блок, який керує чергами переривань (хто головніший: таймер чи натиснута кнопка?).
*   **FLASH:** Регістри для керування пам'яттю (наприклад, налаштування затримки читання, коли ми розганяємо процесор до 72 МГц).
*   **PWR (Power Control):** Регістри для режимів сну (економія батарейки).

#### 2. Таймери (Рахування часу)
*   **TIM1, TIM2, TIM3...:** Це окремі блоки зі своїми регістрами. Вони вміють рахувати мікросекунди, створювати ШІМ (PWM) для керування яскравістю світлодіода або швидкістю двигуна.

#### 3. Інтерфейси зв'язку (Розмови з іншими пристроями)
*   **USART (UART):** Регістри для передачі тексту на комп'ютер.
*   **SPI / I2C:** Регістри для роботи з датчиками, дисплеями та пам'яттю.

#### 4. Аналогові блоки
*   **ADC (Analog-to-Digital Converter):** Регістри, які перетворюють реальну напругу (наприклад, з фоторезистора) у цифри від 0 до 4095.

### Порада для конспекту:
Намалюйте "Глобальну карту пам'яті" (Memory Map). Вона виглядає як одна велика вертикальна вежа, де:
1.  Знизу (**0x0800 0000**) — ваша програма (FLASH).
2.  Вище (**0x2000 0000**) — ваші змінні (SRAM).
3.  Ще вище (**0x4000 0000**) — територія периферії, де живуть RCC, GPIO та всі інші блоки, які ми обговорили.

Це і є весь світ STM32. Ви зараз "заселяєтесь" у район `0x4001 0800` (GPIO) та `0x4002 1000` (RCC).
Це чудове прагнення! Раз ви налаштовані розібратися в усьому детально, давайте розширимо вашу "карту світу" STM32. 

Коли ви програмуєте "на регістрах", ви фактично подорожуєте адресним простором. Ось як виглядає повна ієрархія того, де живуть ваші команди:

### 1. Глобальна карта пам'яті (The Big Picture)
Процесор ARM Cortex-M3 бачить усе як одну довгу "вулицю" адрес від `0x0000 0000` до `0xFFFF FFFF`.



*   **Flash (0x0800 0000):** Тут лежить ваш код. Коли ви прошиваєте мікроконтролер, дані записуються сюди.
*   **SRAM (0x2000 0000):** Тут живуть ваші змінні (масиви, структури), які змінюються під час роботи.
*   **Peripherals (0x4000 0000):** Це "район заводів". Саме тут знаходяться **RCC** та **GPIO**.

---

### 2. Регістр BSRR (0x10) — Секрет професіоналів
Ви зараз використовуєте **ODR (0x0C)** для мигання: `GPIOC_ODR ^= GPIO_PIN_13;`. Це працює, але в реальних проектах частіше використовують **BSRR (Bit Set/Reset Register)**.

**Чому BSRR кращий за ODR?**
1.  **Атомарність:** Коли ви робите `^=` (XOR) з ODR, процесор мусить: *прочитати* значення, *змінити* його в умі і *записати* назад. Якщо в цей момент станеться переривання, може виникнути помилка.
2.  **Швидкість:** **BSRR** дозволяє змінити пін за одну дію запису, не читаючи нічого.



**Як він влаштований:**
*   **Нижні 16 біт (0-15):** Якщо записати "1" у 13-й біт — пін 13 стане **High** (3.3V).
*   **Верхні 16 біт (16-31):** Якщо записати "1" у 29-й біт ($16 + 13$) — пін 13 стане **Low** (0V).

**Приклад у коді:**
```c
GPIOC_BSRR = (1 << 13);      // Увімкнути LED (Set)
delay(300000);
GPIOC_BSRR = (1 << (13+16)); // Вимкнути LED (Reset)
```

---

### 3. Регістр IDR (0x08) — Як читати кнопки
Якщо ви захочете не тільки світити, а й реагувати на натискання кнопки, вам знадобиться **IDR (Input Data Register)**.



Це "дзеркало" ваших пінів. Якщо на фізичній ніжці мікроконтролера є напруга — відповідний біт в IDR стає одиницею. Ви просто читаєте цей регістр у циклі `while`.
