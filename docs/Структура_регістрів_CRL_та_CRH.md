Якщо ви зрозумієте логіку регістрів зараз, то будь-який інший мікроконтролер (AVR, ESP32 чи навіть ARM Cortex-A) не буде для вас проблемою.

Найкраще джерело знань — це **Reference Manual (RM0008)** для серії STM32F1. Це "біблія" вашого чипа. Але оскільки він величезний (1000+ сторінок), давайте я поясню структуру регістрів конфігурації так, щоб ви могли читати документацію як відкриту книгу.

### Структура регістрів CRL та CRH

Кожен пін має **4 біти** в регістрі конфігурації. Ці 4 біти діляться на дві групи:
1.  **MODE [1:0]** (2 біти): визначають напрямок (вхід чи вихід) і швидкість.
2.  **CNF [1:0]** (2 біти): визначають тип (Push-pull, Open-drain, Analog, Floating тощо).



Ось як вони виглядають для кожного піна:

| Біти (приклад для PA0) | Назва | Значення для Output (2MHz, Push-Pull) |
| :--- | :--- | :--- |
| **[1:0]** | **MODE0** | `10` (це 2 у десятковій системі — швидкість 2MHz) |
| **[3:2]** | **CNF0** | `00` (це General purpose output push-pull) |

Разом вони дають `0010`, що дорівнює **0x2** у шістнадцятковій системі. Саме тому ви бачите в коді `(0x2 << 0)`.

---

### Чому для PA0 ми беремо CRL?

Регістри працюють як "поверхи" в будинку:
*   **CRL (Offset 0x00):** обслуговує піни з 0 по 7.
*   **CRH (Offset 0x04):** обслуговує піни з 8 по 15.

Якщо ви хочете налаштувати **PA0**, ви берете `GPIOA_BASE + 0x00`. Якщо **PA13** — `GPIOA_BASE + 0x04`.



### Як розширювати код для портів A та B?

Ось повна карта адрес для вашого коду:

1.  **Базові адреси (будинки):**
    *   `GPIOA: 0x40010800`
    *   `GPIOB: 0x40010C00`
    *   `GPIOC: 0x40011000`

2.  **Зміщення (кімнати):**
    *   `CRL: + 0x00` (піни 0-7)
    *   `CRH: + 0x04` (піни 8-15)
    *   `IDR: + 0x08` (Input Data — читання)
    *   `ODR: + 0x0C` (Output Data — запис)

### Що почитати (покроково):
1.  **Reference Manual (RM0008):** Розділ **"General-purpose and alternate-function I/Os (GPIO)"**. Там є таблиці, які показують, які саме біти куди ставити.
2.  **Datasheet на конкретний чип (наприклад, STM32F103C8):** Там дивіться розділ **"Memory Map"**. Саме звідти беруться цифри `0x40011000` тощо.
3.  **Статті по "Bare Metal STM32":** Шукайте саме цей термін. Це програмування без бібліотек HAL чи LL, як ви зараз і робите.

**Важливий нюанс про тактування (RCC):**
Якщо ви додаєте Port A, не забудьте "увімкнути йому світло" в регістрі `RCC_APB2ENR`. Для Port A це біт №2.
`RCC_APB2ENR |= (1 << 2);`
