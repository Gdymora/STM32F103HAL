I2C — складніший протокол ніж SPI. Коротко теорія:

```
SPI:  master керує CLK, окремий CS для кожного пристрою
I2C:  тільки 2 дроти, кожен пристрій має адресу
      SCL — clock
      SDA — дані (bidirectional!)
```

**Підключення MPU6050 → Blue Pill:**

```
Blue Pill    MPU6050
─────────────────────
PB6    →     SCL
PB7    →     SDA
3.3V   →     VCC
GND    →     GND
GND    →     AD0  (адреса 0x68)
```

`WHO_AM_I: 0x68` — MPU6050 знайдений і відповідає.

Дані акселерометра змінюються — сенсор живий. `0xCF..` це від'ємне число (старший біт 1) — значить вісь X зараз в мінусі, що нормально залежно від орієнтації плати.

Переведемо в нормальні числа:

```c
int16_t ax = (int16_t)((hi << 8) | lo);
// ax / 16384.0 = g (при ±2g режимі)
// 0xCFD0 = -12336 → -12336/16384 ≈ -0.75g
```

 важливі моменти:

**I2C відрізняється від SPI:**
```
SPI  — відправив і забув, slave не підтверджує
I2C  — кожен байт підтверджується ACK від slave
      без ACK — помилка на шині
```

**Repeated Start** — фішка I2C:
```
START → addr+W → reg → REPEATED START → addr+R → читаємо
```
Без repeated start треба робити STOP між записом і читанням — деякі пристрої це не люблять.

**`(void)I2C1_SR2`** — читання SR2 скидає ADDR прапорець. Якщо забути — I2C зависне.
