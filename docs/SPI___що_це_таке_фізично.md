## SPI — що це таке фізично

SPI (Serial Peripheral Interface) — синхронний протокол. На відміну від UART де є baud rate і кожна сторона рахує час сама, тут є **окремий дріт тактування (SCK)**. Майстер диктує темп, слейв просто слухає.

4 дроти:
- **SCK** — тактовий сигнал від майстра
- **MOSI** — Master Out Slave In (майстер пише)
- **MISO** — Master In Slave Out (слейв відповідає)
- **NSS/CS** — Chip Select, активний LOW — "я звертаюсь до тебе"

Nokia 5110 — тільки приймає, MISO не використовується. Плюс два додаткові GPIO: **RST** і **D/C** (data/command).

## Як виглядає передача одного байта

```
NSS  ‾‾‾‾‾\___________________________/‾‾‾‾‾
SCK  ______/‾\_/‾\_/‾\_/‾\_/‾\_/‾\_/‾\_/‾\__
MOSI ------[7][6][5][4][3][2][1][0]----------
```

NSS тягнеш LOW → надсилаєш 8 біт старшим вперед (MSB first) → NSS назад HIGH. SCK генерує SPI периферія автоматично поки ти пишеш байт в DR.

## Регістри SPI1 на STM32F103

Базова адреса: `0x40013000`

| Регістр | Offset | Призначення |
|---------|--------|-------------|
| SPI_CR1 | 0x00   | головне керування |
| SPI_CR2 | 0x04   | DMA, переривання |
| SPI_SR  | 0x08   | статус |
| SPI_DR  | 0x0C   | дані (читати/писати) |

### SPI_CR1 — найважливіший

```
Біт 15: BIDIMODE  — 0 = 2-лінійний (повний дуплекс)
Біт 11: DFF       — 0 = 8-bit фрейм, 1 = 16-bit
Біт  9: SSM       — Software Slave Management (ми самі керуємо NSS через GPIO)
Біт  8: SSI       — якщо SSM=1, цей біт = логічний стан NSS "всередині"
Біт  7: LSBFIRST  — 0 = MSB first (нам треба саме це)
Біт  6: SPE       — SPI Enable (вмикаємо останнім!)
Біт 5-3: BR[2:0]  — Baud Rate = APB2 / 2^(BR+1)
Біт  2: MSTR      — 1 = Master mode
Біт  1: CPOL      — тактова полярність
Біт  0: CPHA      — фаза захоплення
```

### CPOL і CPHA — режими SPI

Це найбільш заплутана частина. Nokia 5110 (PCD8544) використовує **Mode 0**: CPOL=0, CPHA=0.

```
Mode 0 (CPOL=0, CPHA=0):
- SCK в спокої = LOW
- дані захоплюються по ПЕРЕДНЬОМУ фронту (LOW→HIGH)
- дані змінюються по ЗАДНЬОМУ фронту

SCK: ___/‾\_/‾\_/‾\___
         ↑   ↑   ↑
      захоплення тут
```

Є ще Mode 3 (CPOL=1, CPHA=1) де SCK в спокої HIGH — але для 5110 не потрібен.

### SPI_SR — статус

```
Біт 7: BSY   — 1 = SPI зараз щось передає (чекай!)
Біт 1: TXE   — 1 = TX буфер порожній (можна писати наступний байт)
Біт 0: RXNE  — 1 = є дані в RX буфері
```

### Baud Rate розрахунок

SPI1 сидить на APB2 — 8MHz (як і USART1 на Blue Pill без PLL).

```
BR[2:0] = 0b000 → fPCLK/2   = 4 MHz
BR[2:0] = 0b001 → fPCLK/4   = 2 MHz
BR[2:0] = 0b010 → fPCLK/8   = 1 MHz  ← Nokia 5110 max ~4MHz, будь-що підійде
BR[2:0] = 0b111 → fPCLK/256 = 31 kHz
```

## Піни SPI1 і AF

На STM32F103 SPI1 пов'язаний з конкретними пінами:

Blue Pill    Nokia 5110
─────────────────────────
PA5    →     CLK
PA7    →     DIN (MOSI)
PA4    →     CE  (NSS/CS)
PA3    →     DC  (керуємо вручну)
PA2    →     RST (керуємо вручну)
3.3V   →     VCC
GND    →     GND
3.3V   →     BL

Режим AF Push-Pull в CRL — це `0xB` (той самий що ти вже використовуєш для UART TX).

## Алгоритм ініціалізації

```
1. RCC: вмикаємо тактування SPI1 (APB2ENR біт 12) + GPIOA
2. GPIO: PA5, PA7 → AF Push-Pull (0xB)
         PA4 (NSS), PA2 (D/C), PA3 (RST) → Output (0x2)
3. SPI_CR1: налаштовуємо BR, MSTR=1, SSM=1, SSI=1, CPOL=0, CPHA=0
4. SPE=1 — вмикаємо SPI
5. NSS = HIGH (не вибраний)
```

 вже Nokia 5110 поверх нього.

 Спочатку теорія мінімум:
 
SPI — 4 дроти:
  SCK  — тактовий сигнал (master генерує)
  MOSI — дані від master до slave (DIN на Nokia)
  MISO — дані від slave до master (Nokia не використовує)
  NSS  — chip select (вибираємо пристрій)

Nokia 5110 додатково:
  DC   — 0=команда, 1=дані (пікселі)
  RST  — reset дисплея
Послідовність роботи з Nokia:
1. RST LOW → HIGH  — скидаємо дисплей
2. DC LOW          — режим команди
3. SPI відправляємо байт команди
4. DC HIGH         — режим даних
5. SPI відправляємо байти пікселів
