**Не кожен пін може виводити PWM** — тільки ті що мають Alternate Function прив'язку до таймера.

```
PC13 — звичайний GPIO, немає AF до жодного таймера
PA0  — має AF: TIM2_CH1
PA1  — має AF: TIM2_CH2
PA2  — має AF: TIM2_CH3
PA3  — має AF: TIM2_CH4
```

Це визначено в **datasheet STM32F103** — таблиця Alternate Function mapping. Не RM0008 а саме datasheet.

Метафорично:

> «PC13 — це звичайна розетка. PA0 — розетка з прямим дротом до електростанції (TIM2). Можна подавати напругу на обидві, але тільки одна підключена до таймера який може автоматично перемикати її 1000 разів на секунду.»

Коли ми пишемо:
```c
TIM2_CCER = (1 << 0);  // CC1E — увімкнути CH1
```

Апаратно TIM2 починає керувати піном PA0 **сам** — без участі процесора. Саме тому PWM не навантажує CPU.
