## 🔌 Підключення до комп'ютера

### Перше підключення (USB RNDIS)

Плата підключається через USB і прикидається Ethernet адаптером.  
Linux бачить її як інтерфейс `enx...` — назва генерується з MAC адреси.

**Крок 1 — знайти інтерфейс:**
```bash
ip addr show | grep "enx"
# Шукаємо enx... — це завжди плата
# Приклад: enx0a400188dd9f або enx5a11342ec4ad
# Всі інші інтерфейси мають стандартні імена: lo, enp0s25, docker0, virbr0
```

**Крок 2 — призначити IP на комп'ютері:**
```bash
sudo ip addr add 172.32.0.100/24 dev enx5a11342ec4ad
sudo ip link set enx5a11342ec4ad up
# /24 = маска 255.255.255.0 — означає що обидва пристрої в одній підмережі
```

**Крок 3 — знайти IP плати:**
```bash
sudo nmap -sn 172.32.0.0/24 -e enx5a11342ec4ad
# -sn = тільки ping scan, без сканування портів
# Дефолтний IP з документації 172.32.0.70 — не вір, скануй завжди
```

### Рівень 3 — статичний IP на самій платі ✅

**Найкраще рішення** — плата завжди має однаковий IP незалежно від комп'ютера.

**❌ Що не спрацювало — `/etc/network/interfaces`:**
```bash
cat > /etc/network/interfaces << 'EOF'
# interface file auto-generated by buildroot
auto lo
iface lo inet loopback

auto eth0
iface eth0 inet static
    address 192.168.1.102
    netmask 255.255.255.0
    gateway 192.168.1.1
    pre-up killall udhcpc 2>/dev/null; true
EOF
```
> `pre-up killall udhcpc` — логічно мало б вбивати udhcpc перед підняттям інтерфейсу,  
> але на Buildroot `udhcpc` стартує **незалежно від ifup** і просто приходить знову після хука.  
> В результаті — після ребуту плата отримує `.101` від роутера замість нашого `.102`.

**✅ Правильне рішення — S99staticip:**  
Створюємо init.d скрипт з найвищим пріоритетом (S99 = виконується останнім).  
Він приходить вже після `udhcpc` і мовчки переписує IP.

```bash
cat > /etc/init.d/S99staticip << 'EOF'
#!/bin/sh
case $1 in
start)
    sleep 5
    ifconfig eth0 192.168.1.102 netmask 255.255.255.0
    route del default 2>/dev/null
    route add default gw 192.168.1.1
    ;;
stop) ;;
*) exit 1 ;;
esac
EOF
chmod +x /etc/init.d/S99staticip
```

> ℹ️ `sleep 5` — чекаємо поки `udhcpc` завершить роботу, тільки потім перезаписуємо IP.  
> Так, це брутально. Але це єдиний надійний спосіб на Buildroot Luckfox.

**Перевірити після ребуту:**
```bash
ip addr show eth0
# Шукаємо: inet 192.168.1.102/24 ... valid_lft forever

ip route show
# Має бути: default via 192.168.1.1 dev eth0

ping -c3 192.168.1.1   # роутер
ping -c3 8.8.8.8       # інтернет
```

> ⚠️ **Важливо:** після `reboot` якщо кабель не підключений фізично під час старту —  
> S99staticip все одно виставить IP, але `NO-CARRIER` може заважати маршрутизації.  
> Якщо щось не так: `ifdown eth0 && ifup eth0`

**Перевірити що udhcpc не заважає:**
```bash
ps aux | grep udhcpc
# Якщо крутиться після S99staticip — вбити: kill $(pgrep udhcpc)
```

> ℹ️ **Чому плата зникла зі списку DHCP клієнтів роутера?**  
> Зі статичним IP плата більше не робить DHCP запит.  
> Роутер бачить трафік але не реєструє в таблиці клієнтів — це нормально.

## 🩺 Діагностика мережі — алгоритм

Щось не працює? Йди по кроках зверху вниз — знайдеш де обривається.

### Крок 1 — фізика
```bash
ip link show eth0
# Шукаємо: LOWER_UP — кабель підключений і link є
# Якщо NO-CARRIER — проблема фізична: кабель, роз'єм, порт роутера
```
> ⚠️ Перший підозрюваний завжди кабель — пошевели роз'єми!

### Крок 2 — IP адреса
```bash
ip addr show eth0
# Має бути: inet 192.168.1.102/24 ... valid_lft forever
# Якщо немає IP — ifdown eth0 && ifup eth0
# Якщо IP інший (наприклад .101) — udhcpc перебив статичний
```

### Крок 3 — таблиця маршрутів
```bash
ip route show
# Має бути:
# default via 192.168.1.1 dev eth0   ← вихід в інтернет
# 192.168.1.0/24 dev eth0            ← локальна мережа
# Якщо немає default — ip route add default via 192.168.1.1 dev eth0
```

### Крок 4 — роутер
```bash
ping -c3 192.168.1.1
# Якщо не пінгується — проблема між платою і роутером
# Перевір: кабель, порт роутера, IP в одній підмережі
```

### Крок 5 — інтернет
```bash
ping -c3 8.8.8.8
# Якщо роутер пінгується але 8.8.8.8 ні — проблема з gateway або DNS
```

### Крок 6 — сервіс
```bash
# Чи запущений сервер?
ps aux | grep http-server

# Чи слухає порт?
netstat -tln | grep 8080

# Перевірити локально на платі
wget -O- http://127.0.0.1:8080/api/stats

# Перевірити з мережі
wget -O- http://192.168.1.102:8080/api/stats
```

### Гарячі команди — швидка діагностика

## 🐹 Go — крос-компіляція для Luckfox

> ✅ Перевірено: Hello World + HTTP сервер + JSON API успішно запущені на платі

### Чому Go а не Node.js?

| | Node.js | Go |
|---|---|---|
| Runtime розмір | ~50MB | 0 (статичний бінарник) |
| RAM при старті | ~40MB | ~2MB |
| Крос-компіляція | складна | одна команда |
| Є в Buildroot | ні | не потрібен |

> ℹ️ `statically linked` — Go компілює всі залежності в один бінарник.  
> Копіюєш один файл на плату і він просто працює. Без `apt install`, без `.so` бібліотек.

---

### Крок 1 — Підготовка Dockerfile

```bash
# Використовуємо Dockerfile.go-luckfox на базі Alpine
# Alpine замість Debian Bookworm — суттєво менше вразливостей
docker build -f Dockerfile.go-luckfox -t go-arm-dev .

# Перевірити що образ є
docker images | grep go-arm-dev
```

**Dockerfile.go-luckfox:**
```dockerfile
FROM golang:1.23-alpine

RUN apk add --no-cache \
    git curl vim bash \
    iputils net-tools procps \
    openssh-client file

WORKDIR /go-dev
CMD ["tail", "-f", "/dev/null"]
```

---

### Крок 2 — Запуск контейнера

```bash
# З volume — папка проєкту доступна всередині як /go-dev
docker run -it --rm -v $(pwd):/go-dev go-arm-dev bash
```

---

### Крок 3 — Крос-компіляція

```bash
# Всередині контейнера
cd /go-dev/luckfox
GOOS=linux GOARCH=arm GOARM=7 go build -o http-server server.go

# Перевірити бінарник
file http-server
# → ELF 32-bit LSB executable, ARM, EABI5, statically linked ✅
```

> ℹ️ `GOOS=linux` — цільова ОС  
> ℹ️ `GOARCH=arm` — архітектура ARM  
> ℹ️ `GOARM=7` — версія ARM (Cortex-A7 = ARMv7)

---

### Крок 4 — Копіювання і запуск на платі

```bash
# З комп'ютера (не в Docker)
scp -o PubkeyAuthentication=no ./luckfox/http-server root@192.168.1.102:/root/

# На платі
chmod +x /root/http-server

# Запуск у фоні — консоль залишається вільною
/root/http-server &

# Зупинити
kill $(pgrep http-server)
```

## 🐛 Відомі проблеми і фікси

| Проблема | Причина | Фікс |
|---|---|---|
| `Too many authentication failures` | ssh-agent перебирає ключі | `-o PubkeyAuthentication=no` |
| `OSError: [Errno 97]` в http.server | немає IPv6 на ядрі | додати `-b 0.0.0.0` |
| SCP не копіює файл з кирилицею | unicode в шляху | скопіювати в `/tmp` спочатку |
| Дефолтний IP 172.32.0.70 не пінгується | плата бере інший IP | сканувати через `nmap` |
| `loadavg` постійно ~6.x | NPU/RISC-V MCU kernel threads чекають на hardware | норма для цієї прошивки, `%CPU 0%` — система вільна |
| Samba запущена з коробки | Luckfox прошивка включає smbd/nmbd | `killall smbd nmbd` — звільняє ~3MB RAM |
| http-server VSZ 523MB | Go runtime резервує віртуальний адресний простір | норма, реальна RAM нормальна |
| Після ребуту eth0 без IP | `ifup` спрацьовує до підняття PHY link (`NO-CARRIER`) | підключити кабель до ребуту, або `ifdown eth0 && ifup eth0` |
| udhcpc перебиває статичний IP після ребуту | Buildroot запускає udhcpc до будь-якого нашого конфігу, `/etc/network/interfaces` не допомагає | Використати `/etc/init.d/S99staticip` — скрипт з `sleep 5` + `ifconfig` що приходить після udhcpc |
| Плата зникла з DHCP списку роутера | статичний IP не реєструється в DHCP таблиці | норма, трафік ходить, просто не видно в списку |
| Нестабільний ping (53ms, 715ms, 759ms) | поганий контакт кабелю або роз'єму | замінити кабель або пошевелити роз'єми |
| `netstat -tlnp` не працює | BusyBox версія не підтримує `-p` | використовувати `netstat -tln` без `-p` |
| Камера не розпізнається, rkipc.ini не з'являється | прошивка не містить IQ файл для сенсора | перепрошити новим образом з підтримкою потрібного сенсора |
| SSH: "WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED" | після перепрошивки плата генерує нові SSH ключі | `ssh-keygen -f ~/.ssh/known_hosts -R "172.32.0.93"` |
| Два IP одночасно після `ifup` | udhcpc встиг отримати IP до `pre-up killall` | `kill $(pgrep udhcpc)` + `ip addr del 192.168.1.101/24 dev eth0` + `ifdown eth0 && ifup eth0` |
