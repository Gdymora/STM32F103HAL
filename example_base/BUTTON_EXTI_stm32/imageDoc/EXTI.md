# 📒 Конспект: Зовнішні переривання (EXTI) в STM32

## 1. Основна концепція
**Переривання** — це подія, яка змушує процесор кинути основну роботу в `main()` і миттєво виконати спеціальну функцію-обробник (**ISR**).
**EXTI** (External Interrupt) — модуль, що стежить за ніжками (GPIO) і створює ці переривання.

---

## 2. Шлях сигналу (Ланцюжок налаштування)
Щоб кнопка "ожила", сигнал має пройти через чотири блоки:

### А. Блок GPIO (Вхід)
Ми кажемо ніжці, що вона — вхід.
*   **У коді:** `GPIOA_CRL |= (0x8 << 4);` (Вхід з підтяжкою для Pin 1).
*   **Суть:** Фізично підключаємо кнопку до мікроконтролера.

### Б. Блок AFIO (Маршрутизатор)
Оскільки вхідних ніжок багато (PA1, PB1, PC1...), а лінія EXTI1 — одна, AFIO працює як перемикач (мультиплексор).
*   **У коді:** `AFIO_EXTICR1 &= ~(0xF << 4);`
*   **Суть:** Кажемо лінії EXTI1: "Слухай саме Port A".


### В. Блок EXTI (Детектор події)
Тут ми налаштовуємо, на що саме реагувати.
*   **IMR (Mask):** Дозволяємо переривання на цій лінії (`1 << 1`).
*   **FTSR (Falling):** Реагуємо на спад напруги (натискання до GND) (`1 << 1`).
*   **Суть:** Налаштовуємо "датчик" на лінії.

### Г. Блок NVIC (Головний менеджер)
NVIC керує всіма перериваннями в системі. Кожне джерело має свій номер (**IRQ**).
*   **EXTI1** має фіксований номер **7**.
*   **У коді:** `NVIC_ISER0 |= (1 << 7);`
*   **Суть:** Відчиняємо двері в процесор для сигналу №7.

---

## 3. Таблиця зв'язків (Шпаргалка для зошита)

Якщо ти міняєш кнопку з одного піна на інший, користуйся цією таблицею:

| Пін | Лінія EXTI | Регістр AFIO | Біт у EXTI | IRQ (NVIC) | Назва функції |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Px0** | EXTI0 | EXTICR1 [3:0] | `1 << 0` | **6** | `EXTI0_IRQHandler` |
| **Px1** | **EXTI1** | **EXTICR1 [7:4]** | `**1 << 1**` | **7** | `**EXTI1_IRQHandler**` |
| **Px2** | EXTI2 | EXTICR1 [11:8] | `1 << 2` | **8** | `EXTI2_IRQHandler` |

---

## 4. Як працює обробник (Код)

Функція переривання **мусить** скинути прапорець, інакше вона буде викликатися по колу.

```c
void EXTI1_IRQHandler(void) {
    // 1. Перевірка: чи справді це лінія 1?
    if (EXTI_PR & (1 << 1)) { 
        
        // --- ТВІЙ КОД ТУТ ---
        
        // 2. Скидання: записуємо 1 у прапорець (це важливо!)
        EXTI_PR = (1 << 1); 
    }
}
```

---

## 5. Чому ми пишемо `(1 << 1)`?
*   Цифра `1` після зсуву `<<` — це **номер піна** і **номер лінії EXTI**.
*   Якщо кнопка на PA**1** → пишемо `1 << 1`.
*   Якщо кнопка на PA**5** → пишемо `1 << 5`.

---

**Порада для розуміння:** Уяви, що EXTI — це охоронець, який стоїть біля дверей. `IMR` — це наказ йому не спати, `FTSR` — прикмети злодія (натискання кнопки), а `PR` — це звіт про подію, який ти маєш підписати (скинути), щоб охоронець повернувся на пост.
